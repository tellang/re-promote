# Default Commit Convention

## Format

```
<type>(<scope>): <summary>

<body>
```

## Rules

- **Language**: 한글 기본. 프로젝트 기존 커밋 언어를 따른다.
- **Summary**: 명령형, 72자 이내, 마침표 없음
- **Body** (선택): 변경 이유와 맥락 설명
- **Footer**: AI 관련 attribution 절대 금지

## Types

| Type | 용도 |
|------|------|
| `feat` | 새로운 기능 |
| `fix` | 버그 수정 |
| `refactor` | 동작 변경 없는 코드 구조 개선 |
| `docs` | 문서 변경 |
| `chore` | 빌드, CI, 의존성, 기타 잡무 |
| `test` | 테스트 추가/수정 |
| `build` | 빌드 시스템 변경 |
| `ci` | CI 설정 변경 |
| `perf` | 성능 개선 |
| `style` | 코드 포맷팅 (동작 무관) |

## Scope (Optional)

괄호 안에 영향 범위를 명시. 예: `feat(auth)`, `fix(api)`, `chore(deps)`

## Examples

```
feat(viewer): 실시간 SSE 웹 뷰어 + 검색 + PDF 내보내기

SSE 기반 실시간 전사 결과 스트리밍과 웹 뷰어를 추가한다.
검색, PDF 내보내기, WCAG AA 접근성을 지원한다.
```

```
fix(server): QA 버그 수정 + 토스트 알림 + 서버 리팩터링
```

```
chore: PyPI 배포 + 배지 설정
```

## Forbidden

- `Co-Authored-By: Claude`, `Co-Authored-By: Anthropic` 등 AI attribution
- `Generated by`, `Assisted by` 등 AI 관련 마커
- 영어/한글 혼용 summary (하나로 통일)
